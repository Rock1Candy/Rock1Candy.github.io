如何手动配置WPS加载项功能区？（没写完）

嗨害，朋友们大家好，我是练习时长两年半的业余编程爱好者。只学过C，编程经验仅限hello world。

分享一个WPS加载项的开发的小经验： 手动配置Excel加载项功能区，瞎整试出来的。

现在我们来看如何开发WPS的加载项。Excel的加载项也是.xlam为扩展的一个文件，同样默认没有图标，本文中，将使用Custom UI Editor配置Excel加载项的功能区。

那么，如何让WPS的Excel加载项也能有可见的图标呢，下面以Excel加载项为例说明，简单而直接。

1.准备工作

安装WPS。2020及以后版本。

准备Custom UI Editor。最好选择支持中文版的。待会要用来配置工具栏。

下载并解压wpsjs。在wpsjs这个包里找到？这几个文件并打开，待会要用里面的代码。

2.改改代码

新建一个Excel表格，打开开发工具。

新建(插入)一个模块，命名为Code4Custom，其他名字也可以啦，whatever，把ribbon.xml里面的代码复制过来，先不要改。这并不是js代码，把它复制过来只是方便查看。

[图片]

这里面的内容就是工具栏里的功能，如图这里有5个按钮，这里的button类似表格里的控件，点一下就触发一个事件。
OnAction就是指定的是什么事件，我们是手动配置，不能用枚举控制，所以要给每个按钮都指定OnAction事件.

[图片]

然后再新建一个模块，复制粘贴ribbon.js里面的代码。ribbon.js里定义了事件的监听，触发时运行什么代码。把OnAddinLoad里面的东西全删了，下面的代码不用管它就行，都是要把每个function里面的return true保留，否则按钮是灰的。完成后另存为.xlam。
[图片]

3.配置功能区

启动Custom UI Editor，打开刚刚的.xlam，在右侧粘贴ribbon.xml里面的代码，然后先后点击validate、save。

到这里就配置完成了，可以尝试加载刚刚的xlam文件，之后就可以按需要在WPS宏编辑器里编写自己需要的功能了，然后把function的名字粘贴到Custom UI Editor里面的OnAction就可以了。

此外，还可以按自己需要选择按钮的UI图标，在getImage=””填入图标代号就OK了。
